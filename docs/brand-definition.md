# Anemonet ブランド定義書

> 本ドキュメントは Anemonet のブランドアイデンティティを定義する。
> サイトコピー・デザイン・コミュニケーションすべての基盤となる。

---

## 1. ブランド名の由来

**Anemonet** = **Anemone（アネモネ）** + **net（ネットワーク）**

### アネモネの象徴

| 特性 | 象徴 | ブランドへの意味づけ |
|------|------|---------------------|
| 語源「Anemos（風）」 | 風の娘、風の花 | 変化の風を読み、時代に適応し続ける |
| 早春にいち早く咲く | 先駆け、始まりの花 | 新しい技術・手法をいち早く取り入れ、先駆けて動く |
| 朝に開き、夜に閉じる | 復活・再生 | スクラップ＆リビルド — 壊して学び、再び立ち上げる |
| 種は風に運ばれて広がる | 拡散・循環 | 知見がネットワークを通じて循環し、価値が広がる |
| 花びらに見えるものは「がく」 | 本質は表面と異なる | 表面的な要件の奥にある本質を見抜く |
| 白の花言葉：真実・希望・期待 | 誠実さ | 顧客が本当に求めているものを引き出す |
| 紫の花言葉：守護・期待 | ブランドカラーとの一致 | 顧客の事業を守り育てる |
| 多彩な花色 | 多様性 | 幅広い顧客・多様な事業領域に対応 |

### 「net」の多層的な意味

| 層 | 意味 |
|----|------|
| Network | 人と人、知見と知見をつなぐハブ |
| Internet / System | システム開発という事業ドメイン |
| Safety Net | 顧客の事業を支える安全網としての伴走 |

---

## 2. Mission（使命）

> **風を読み、本質を咲かせる。**

変化の激しいシステム開発の世界で、時代の風を読み続け、
顧客が言葉にできていない本当のニーズを引き出し、最適な形で実現する。

---

## 3. Vision（目指す姿）

> **技術と人をつなぎ、知見が循環するネットワークの中心であり続ける。**

アネモネの種が風に乗って広がるように、プロジェクトを通じて得た知見を蓄積・循環させ、
関わるすべての人に価値を届ける。

---

## 4. Values（行動指針）

| Value | 由来 | 意味 |
|-------|------|------|
| **Wind Reading（風を読む）** | Anemos＝風 | 技術の潮流を常に捉え、手法を更新し続ける |
| **Root Finding（根を見つける）** | がくの下の本質 | 表面の要件ではなく、潜在的な課題の根に辿り着く |
| **Rebuild Courage（壊す勇気）** | 朝に咲き夜に閉じる再生 | うまくいかなければ壊して学び、より良く作り直す |
| **Grow Together（共に咲く）** | 群生する花 | 外注先ではなく、同じチームの一員として伴走する |
| **Spread Seeds（種を蒔く）** | 風で種が広がる | 得た知見をネットワークに還元し、共有する |

---

## 5. Tagline

**メイン**: 風を読み、本質を咲かせる。

**サブ（サービス説明用）**: AIの力で、本当に求められるシステムを。

## 6. Brand Personality（ブランドの人格）

### キャラクター：「風を読む庭師」

- 季節の変わり目をいち早く感じ取り、種を蒔く準備ができている
- 寡黙な職人ではなく、顧客と対話しながら「何を育てたいのか」を一緒に探る
- 一つの庭（プロジェクト）で得た学びを、次の庭にも活かす
- 自分の手法に固執せず、季節（時代）に合わせて道具を変える柔軟さ

### トーン

- 穏やかだが芯がある
- 誠実で、飾りすぎない
- 専門性に裏打ちされた安心感

### 避けたい印象

- 安売り感、何でも屋感
- テック感が強すぎる（AI推し一辺倒）
- 大企業的な堅さ・形式主義
- 冷たさ・機械的な印象

---

## 7. Brand Story

アネモネは「風の花」。ギリシャ語で風を意味する「Anemos」を語源に持ち、
早春の風が吹くといち早く花を開く。

Anemonet は、その名の通り、変化の風を読み取り、いち早く動く。

顧客が本当に必要としているものは、最初から明確とは限らない。
アネモネの花びらに見える部分が実は「がく」であるように、
表面の要件の奥には、まだ言葉になっていない本質がある。

私たちは、動くシステムを起点に対話を重ね、つくっては壊し、その本質に辿り着く。

そしてアネモネの種が風に乗って広がるように、
一つのプロジェクトで得た知見を次へとつなぎ、関わるすべての人に価値を届ける。

風を読み、本質を咲かせる。それが Anemonet の在り方。

## 8. 差別化ポジション

| 一般的なフリーランス / SIer | Anemonet |
|-----------------------------|----------|
| 言われたものを作る | 本質を引き出してから作る |
| 一つの技術スタックに特化 | 風を読み、手法を常にアップデート |
| 納品して終わり | 同じチームとして伴走し、知見を蓄積 |
| プロジェクト単位の関係 | ネットワークとして継続的につながる |

## 9. Visual Identity

### カラー

- **Primary（紫）**: `#5C38C0` — アネモネの紫。守護・期待の花言葉と一致
- **Accent**: ブランドの多彩さを表現するアクセントカラー
- **Neutral**: 11段階のグレースケール

### ビジュアルトーン

**Elegant Modern — "Organic Precision"（有機的精密さ）**

- 花の有機的な曲線と、システムの精密さの共存
- 自然の温かみ × 技術の確実さ

### モチーフ

- アネモネの花弁（4枚〜5枚の花弁を抽象化）
- 風の流れ（曲線・グラデーション）
- 種の拡散（ネットワーク的な広がり）

## 10. サービス体系

### サービス 1: AI駆動開発

**Iterative Prototyping with AI**

動くシステムを起点に対話を重ね、つくっては壊し、本質に辿り着く開発手法。
生成AIを活用した高速イテレーションで、仮説検証のサイクルを圧倒的に短縮する。

**Values との接続**:
- Root Finding: 潜在ニーズを動くプロトタイプで引き出す
- Rebuild Courage: スクラップ＆リビルドで技術的負債を持ち越さない
- Wind Reading: AI 技術の進化を常に取り込む

### サービス 2: Claude Code 開発基盤構築

**AI-Assisted Development Infrastructure**

Claude Code の導入を検討する開発チーム向けに、
プロジェクトの技術スタックに最適化された開発基盤を設計・構築する。

**Values との接続**:
- Spread Seeds: 得た知見を基盤として体系化し、チーム全体に還元
- Grow Together: 導入して終わりではなく、チームに伴走
