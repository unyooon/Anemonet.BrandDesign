Philosophy

Anemonet とは

アネモネは「風の花」。ギリシャ語で風を意味する Anemos を語源に持ち、 早春の風が吹くといち早く花を開きます。

Anemonet は、その名の通り、変化の風を読み取り、いち早く動きます。

顧客が本当に必要としているものは、最初から明確とは限りません。 アネモネの花びらに見える部分が実は「がく」であるように、 表面の要件の奥には、まだ言葉になっていない本質があります。

私たちは、動くシステムを起点に対話を重ね、つくっては壊し、 その本質に辿り着きます。

そしてアネモネの種が風に乗って広がるように、 一つのプロジェクトで得た知見を次へとつなぎ、 関わるすべての人に価値を届けます。

Mission

「風を読み、本質を咲かせる。」

変化の激しいシステム開発の世界で、時代の風を読み続け、顧客が言葉にできていない本当のニーズを引き出し、最適な形で実現する。

Vision

「技術と人をつなぎ、知見が循環するネットワークの中心であり続ける。」

アネモネの種が風に乗って広がるように、プロジェクトを通じて得た知見を蓄積・循環させ、関わるすべての人に価値を届ける。

AI駆動開発

Iterative Prototyping with AI

仕様書を書き上げてから開発する従来型では、完成後に「思っていたものと違う」が起きがちです。かといって要件が曖昧なままAIで高速に作り続ければ、コードは膨れ運用コストだけが増えていく。

Anemonetの提案するAI駆動開発は「作っては壊す」が前提。プロトタイプで要件の本質を見極めてから、本番品質で構築し直す。速さと堅牢さを両立します。

1

作って見せる

AIで動くプロトタイプを即座に構築し、画面を見ながら一緒に方向性を決める。

2

壊して学ぶ

得た知見をもとにゼロから再設計。負債を持ち越さない。

3

本番品質で仕上げる

要件が固まった段階で、堅牢なアーキテクチャで本開発。

1〜2 を高速に反復し、要件の本質を見極める

Claude Code 開発基盤構築

AI-Assisted Development Infrastructure

Claude Code の導入を検討している開発チーム向けに、プロジェクトの技術スタックに最適化された開発基盤を設計・構築します。 ツールを入れるだけでは得られない、チーム全体の開発生産性を引き上げる仕組みを整えます。

CLAUDE.md 設計

プロジェクトのアーキテクチャ・コーディング規約・ワークフローに沿った指示書を構築。AI が一貫した品質でコードを生成する土台を整備します。
